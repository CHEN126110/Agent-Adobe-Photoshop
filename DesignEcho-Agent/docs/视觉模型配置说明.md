## 使用场景推荐

### 1. 设计分析（design analysis）

**首选：** `google-gemini-3-flash` 或 `ollama-cloud-qwen3-vl`

原因：
- 需要理解设计构图、配色、排版
- 需要中文输出分析结果
- 成本合理，响应快

### 2. 审美决策验证（aesthetic validation）

**首选：** `openrouter-claude-3.5-sonnet` 或 `google-gemini-3-pro`

原因：
- 需要深度推理能力
- 需要对比多个设计方案
- 需要给出详细的决策理由

### 3. 本地快速预览（local preview）

**首选：** `local-llava-7b` 或 `local-minicpm-v-8b`

原因：
- 无需联网
- 响应快
- 适合频繁调用

## 配置方式

### 方法一：通过设置界面

1. 打开设置面板
2. 选择「模型配置」
3. 在「视觉分析模型」下拉框选择

### 方法二：通过 API

```typescript
// 设置视觉模型
await window.electron.invoke('vlm:setVisionModel', 'google-gemini-3-flash');

// 分析设计
const result = await window.electron.invoke('vlm:analyzeDesign', {
    imageBase64: '...',
    depth: 'standard',
    aspects: ['strengths', 'improvements', 'trends']
});
```

## 注意事项

1. **API Key 配置**
   - 云端模型需要在设置中配置对应的 API Key
   - Google 模型需要 `google` API Key
   - OpenRouter 模型需要 `openrouter` API Key
   - Ollama Cloud 需要 `ollamaApiKey`

2. **图片格式**
   - 支持 JPEG、PNG、WebP
   - 建议图片不超过 4MB
   - Base64 格式传输

3. **本地模型**
   - 需要先安装 Ollama
   - 需要拉取对应模型：`ollama pull llava:13b`
   - 确保 Ollama 服务运行中

4. **成本控制**
   - 日常使用推荐 `google-gemini-3-flash`（约 $0.20/百万 tokens）
   - 深度分析使用 `openrouter-claude-3.5-sonnet`
   - 频繁调用考虑本地模型
