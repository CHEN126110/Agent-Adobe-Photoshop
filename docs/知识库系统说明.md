## 系统概述

DesignEcho 知识库系统是一个基于 **RAG (Retrieval-Augmented Generation)** 的设计知识管理平台，主要用于：

- **存储和检索**设计相关知识（卖点、痛点、配色、文案模板等）
- **索引用户作品**（主图、详情页、SKU 等设计文件）
- **增强 AI 对话**，让 Agent 能够基于用户历史作品和专业知识提供更精准的设计建议

### 核心特点

| 特性 | 说明 |
|------|------|
| **本地向量数据库** | 使用 LanceDB，数据完全本地存储，保护用户隐私 |
| **中文语义检索** | 使用 `bge-small-zh-v1.5` 模型，专为中文优化 |
| **多源知识融合** | 支持内置知识、用户导入、自动学习三种来源 |
| **个性化推荐** | 基于设计师档案的个性化检索加权 |
| **文件夹导入** | 支持直接选择项目文件夹自动索引 |

### 1. 用户作品导入流程

```
用户选择文件夹
      │
      ▼
┌─────────────────┐
│ ProjectIndexer  │  ← 扫描项目目录
│ scanProjects()  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 识别文件类型     │  ← 主图/详情页/SKU/PSD
│ (根据目录结构)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ EmbeddingService│  ← 生成文本描述并向量化
│ embed(text)     │    (可选: 使用 VLM 分析图片)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ VectorStore     │  ← 存储到 LanceDB
│ upsert(entries) │
└─────────────────┘
```

### 2. 知识检索流程

```
用户查询 "红色袜子主图"
         │
         ▼
┌─────────────────┐
│ EmbeddingService│  ← 查询向量化
│ embed(query)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ RetrievalEngine │  ← 混合检索

         │
         ▼
┌─────────────────┐
│ 个性化加权       │  ← 基于设计师档案

└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 返回检索结果     │  → 注入到 Agent Prompt
│ contextForPrompt │
└─────────────────┘
```

## 关键服务

### 1. EmbeddingService (向量化服务)

**路径**: `src/main/services/rag/embedding-service.ts`

**功能**:
- 加载本地 ONNX 模型 (`bge-small-zh-v1.5`)
- 文本向量化 (`embed()`)
- 批量向量化 (`embedBatch()`)

**配置**:
```typescript
{
    modelId: 'Xenova/bge-small-zh-v1.5',
    quantized: true,      // 使用量化模型 (更快)
    batchSize: 32,        // 批处理大小
    dimension: 512        // 向量维度
}
```

### 2. VectorStore (向量存储)

**路径**: `src/main/services/rag/vector-store.ts`

**功能**:
- 向量数据的增删改查
- 余弦相似度搜索
- 元数据过滤

**存储位置**: `%APPDATA%/DesignEcho/lancedb/knowledge_vectors`

### 3. ProjectIndexer (项目索引器)

**路径**: `src/main/services/project-indexer.ts`

**功能**:
- 扫描项目目录结构
- 识别文件类型 (主图/详情页/SKU/PSD)
- 生成文件描述
- 批量索引到向量库

**支持的目录结构**:
```
项目文件夹/
├── 主图/           → main_image
├── SKU/            → sku  
├── images/         → detail_page_slice
├── PSD/            → psd
├── 配置文件/       → config (忽略)
└── 模板文件/       → template (忽略)
```

### 4. RetrievalEngine (检索引擎)

**路径**: `src/main/services/rag/retrieval-engine.ts`

**功能**:
- 混合检索 (语义 + 关键词)
- RRF (Reciprocal Rank Fusion) 融合
- 个性化加权

**检索参数**:
```typescript
{
    semanticWeight: 0.6,    // 语义检索权重
    keywordWeight: 0.4,     // 关键词检索权重
    rrfK: 60,               // RRF 参数
    defaultLimit: 10,       // 默认返回数量
    minScore: 0.3           // 最低分数阈值
}
```

## 用户作品导入

### 导入方式

**当前支持**: 📂 **文件夹选择导入**

用户在「知识库」→「导入」页面，点击「选择项目文件夹」，选择包含设计作品的目录即可自动索引。

### 支持的项目结构 (以 C-649 为例)

```
C-649/
├── 主图/                     ← 自动识别为 main_image
│   ├── 800-1.jpg
│   └── 800-2.jpg
├── SKU/                      ← 自动识别为 sku
│   ├── 3双装.jpg
│   └── 5双装.jpg
├── images/                   ← 自动识别为 detail_page_slice
│   ├── 详情页01.jpg
│   └── 详情页02.jpg
├── PSD/                      ← 自动识别为 psd
│   └── 详情页模板.psd
├── 配置文件/                  ← 忽略
├── 模板文件/                  ← 忽略
└── knowledge-manifest.json   ← 可选: 已生成的知识清单
```

### 索引结果

每个文件会生成一条知识记录：

```json
{
  "id": "C-649:主图/800-1.jpg",
  "type": "main_image",
  "text": "main_image: 主图/800-1.jpg (C-649)",
  "title": "C-649 - 主图/800-1.jpg",
  "metadata": {
    "source": "user_project",
    "categories": ["main_image", "C-649"],
    "keywords": ["main-image"],
    "filePath": "D:/A1 neveralone旗舰店/C-649/主图/800-1.jpg",
    "projectId": "C-649"
  }
}
```

### VLM 增强描述 (可选)

勾选「使用视觉模型」后，系统会调用 Gemini 2.0 Flash 分析图片内容，生成更精准的描述：

```
"红色蝴蝶结袜子主图，白底拍摄，产品居中展示，突出蝴蝶结装饰和柔软材质"
```

## RAG 检索机制

### 混合检索策略

1. **语义检索** (权重 60%)
   - 将查询文本向量化
   - 在向量库中进行余弦相似度搜索
   - 返回语义最相近的 N 条结果

2. **关键词检索** (权重 40%)
   - 对查询进行分词
   - 匹配知识条目的 keywords 和 categories
   - 支持布尔查询

3. **RRF 融合**
   - 使用 Reciprocal Rank Fusion 算法合并两种检索结果
   - 公式: `score = Σ 1/(k + rank)`

### 个性化加权

基于「设计师档案」进行个性化调整：

- **偏好类目加权**: 用户常用的类目权重 +20%
- **历史点击加权**: 用户点击过的知识权重 +10%
- **排除主题**: 用户标记不感兴趣的主题自动过滤

## 常见问题

### Q: 向量化速度慢怎么办？

A: 首次加载模型需要约 10-30 秒，之后会缓存。如果仍然慢，检查是否开启了 `quantized: true`。

### Q: 如何清空已索引的知识？

A: 在知识库面板点击「清空用户索引」，或调用 `project:clearIndex` IPC。

### Q: 支持哪些图片格式？

A: 支持 `.jpg`、`.jpeg`、`.png` 格式的图片文件。

### Q: PSD 文件会被索引吗？

A: 目前 PSD 文件只记录元信息（路径、大小），不解析内部结构。未来会支持提取 PSD 内的文本图层和结构信息。
