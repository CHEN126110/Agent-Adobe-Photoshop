# 设计智能体实施蓝图 v1

> 目标：让 DesignEcho 从“工具调用器”升级为“可规划、可执行、可复盘”的设计智能体。
> 覆盖任务：SKU 批量、详情页设计、主图设计。

## 1. 当前问题

1. 执行能力强，但规划能力弱（缺少稳定 Plan）。
2. autoDesign 接口能力声明与实际执行不一致。
3. 图层语义理解依赖命名规则，泛化不足。
4. 文案缺少硬约束（禁词/风格/合规）和后验质检。
5. 评估已有，但没有形成“低分自动重试”的闭环。

## 2. 目标能力

1. 输入：模板 + 素材 + 任务目标。
2. 输出：可解释的设计方案（Plan）+ 可回放执行日志（Execute）+ 评估报告（Critique）。
3. 失败时可回退：规则方案兜底，不出现“无结果”。

## 3. 系统分层

1. Intent 层（任务意图）
- 产物：DesignIntent
- 字段：taskType、platform、constraints、brandTone、outputSpec

2. Plan 层（设计规划）
- 产物：LayoutPlan / CopyPlan / AssetPlan
- 规则：先规划后执行，禁止边执行边重算

3. Execute 层（原子执行）
- 使用 UXP tools 执行原子动作
- 产物：ExecutionReport（success、error、layerDiff）

4. Critique 层（设计评估）
- 输出：score、issues、recommendations
- 触发：当 score < threshold 时，进入最多 N 轮修正

5. Memory 层（偏好学习）
- 记录：用户采纳/拒绝行为、同类任务高分方案

## 4. 核心数据协议（建议）

```ts
interface DesignIntent {
  taskType: 'sku' | 'detail' | 'mainImage';
  projectPath?: string;
  templatePath?: string;
  outputDir?: string;
  brandTone?: string;
  hardConstraints?: Record<string, unknown>;
  softConstraints?: Record<string, unknown>;
}

interface LayoutPlanStep {
  tool: string;
  params: Record<string, unknown>;
  reason: string;
  blocking?: boolean;
}

interface DesignPlan {
  intent: DesignIntent;
  layoutSteps: LayoutPlanStep[];
  copyPlan?: { layerId: number; text: string; role: string }[];
  assetPlan?: { layerId: number; imagePath: string; fitMode: string }[];
}
```

## 5. 三大任务落地策略

### 5.1 SKU（优先稳定产出）

1. 组合生成继续走规则 + 色彩和谐评分。
2. 增加“模板适配检测”（占位符数量、命名、导出目录校验）。
3. 增加失败重试策略（模板缺失 → 自动搜索候选模板）。

### 5.2 详情页（优先质量闭环）

1. 保留现有 parse/detect/fill/export 流程。
2. 增加 CopyGuard（禁词、长度、语气、广告感过滤）。
3. 增加低分自动修复回路：诊断分 < 阈值 → 自动执行修正计划。

### 5.3 主图（优先执行稳定性）

1. 统一 smartLayout 协议（action + targetBounds）。
2. 引入“主体占比 + 视觉重心”双目标约束。
3. 输出多候选（A/B）并用 Critique 评分择优。

## 6. 文案治理（CopyGuard）

1. 规则层
- 禁词（绝对化/夸张承诺）
- 长度限制（标题/副标题/卖点）
- 标点限制（感叹号、重复符号）

2. 评分层
- 广告感分数
- 可读性分数
- 卖点贴合度分数

3. 执行策略
- 先生成，再过滤，再回写
- 不通过则自动重写最多 2 次

## 7. 图层理解升级

1. 模板入库时建立语义标注（layer role tags）。
2. 运行时优先读取语义标签，命名规则只作回退。
3. 引入“屏级结构完整性检查”：必需角色缺失时阻断执行。

## 8. 分阶段里程碑

### M1（本周）
1. 修复主图链路协议不一致问题。
2. autoDesign 能力声明与行为对齐（不支持即明确报错）。
3. 建立 DesignIntent/DesignPlan 最小结构。

### M2（下周）
1. 详情页接入 CopyGuard。
2. 低分自动修复回路（最多 2 轮）。
3. 主图 A/B 方案输出与自动评分择优。

### M3（第3周）
1. 图层语义标签体系落地。
2. 模板质量门禁（导入阶段）。
3. 统一复盘报告：计划、执行、评分、失败原因。

## 9. 验收指标

1. 任务成功率（SKU/详情页/主图）
- 目标：> 85%

2. 设计评分提升
- before/after 平均提升 > 8 分

3. 文案合规率
- 禁词命中率为 0
- 广告感超阈值比例 < 10%

4. 运行稳定性
- 无结果返回比例 < 2%
- 自动重试后成功恢复比例 > 60%

## 10. 与当前代码的映射

1. Plan/Critique 基础已存在：`design-pipeline.ts`、`design-agent.plan`、`design-agent.diagnose`。
2. Execute 能力已完善：UXP `ToolRegistry` 与各工具分类。
3. 下一步重点是把 Plan 真正作为执行输入，并补足 CopyGuard/语义层。
