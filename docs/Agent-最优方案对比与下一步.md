# Agent 最优方案对比与下一步（2026-03-04）

## 1. 什么叫“最优”
- 稳定：同一需求多次执行，结果一致。
- 准确：选图、选层、置入尺寸、抠图边缘尽量一次到位。
- 可解释：每次关键决策可追溯（为什么选这张图、为什么选这个图层）。
- 可回退：低置信度不直接改，先给候选和确认。
- 可扩展：新增能力（如参考长图模板化）不需要重写整套系统。
- 工程质量：类型安全、边界清晰、可测试、可观测。

## 2. 方案对比

### 方案 A：继续在现有逻辑上打补丁
- 做法：维持当前单层决策 + 工具/技能执行，遇到问题逐点修补。
- 优点：最快上线，改动小。
- 缺点：复杂任务会持续分叉，结果不稳定，技术债继续增长。
- 适用：短期止血。

### 方案 B：双层决策（Planner + Critic）【推荐】
- 做法：保留现有执行器，新增“任务规划层 + 质量审查层”。
- 优点：稳定性、可解释性、可控性显著提升；最适合当前阶段。
- 缺点：改造量中等，需要补充日志与评测。
- 适用：当前主线。

### 方案 C：端到端多模态模型直接驱动
- 做法：让模型直接输出全流程动作并执行。
- 优点：理论上上限高。
- 缺点：可控性差、回归风险高、调试成本高。
- 适用：中长期探索，不建议作为当前主线。

## 3. 结论
- 当前主线选择方案 B。
- 执行策略：先补“稳定性 + 可观测 + 回滚机制”，再加“视觉泛化能力”。

## 4. 后续任务清单

### P0（立即）
- [ ] 重构统一决策层：支持多步计划，而不是一次性决策。
- [ ] 引入置信度门控：低置信度只给候选，不允许直接落盘修改。
- [ ] 统一中英文提示与参数规范，修复乱码文案。
- [ ] 建立可回放日志：输入、候选、评分、最终决策、执行结果。

### P1（核心能力）
- [ ] 新增能力：参考长图 -> 自动模板化（“举一反三”）
  - 输入：用户提供一张长详情页参考图。
  - 输出：可编辑 PSD 模板骨架（按屏拆分，每屏含文案/icon/图片三层组）。
  - 要求：保留结构规则和风格规律，不做机械复制。
- [ ] 参考图语义理解
  - 自动切屏。
  - 模块识别（首屏/KV/面料/痛点/参数/细节等）。
  - 信息层级抽取（主标题、副标题、卖点、辅助元素）。
- [ ] 模板生成器
  - 自动创建父组/子组与命名规范。
  - 生成占位符和安全边界。

### P2（质量闭环）
- [ ] 设计质量评估器（布局一致性、可读性、遮挡、风格偏差）。
- [ ] A/B 策略实验（选图策略、置入策略、裁切策略）。
- [ ] 回归测试集（30-50 套详情页样本，持续回归）。

## 5. 里程碑
- M1（1周）：决策层稳定化 + 可观测日志。
- M2（2周）：长图模板化 MVP（可生成可编辑结构）。
- M3（3-4周）：质量评估闭环 + 自动回归。
