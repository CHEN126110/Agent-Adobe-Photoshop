## 1. 概述

### 1.1 设计目标

模板系统旨在实现：
- **标准化**: 统一的占位符命名和结构
- **可扩展**: 支持自定义模板和占位符
- **智能化**: AI 可理解模板语义并自动填充
- **高效性**: 批量生成时复用模板结构

### 1.2 核心概念

```
┌─────────────────────────────────────────────────────────────────┐
│                      模板系统架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   模板 (Template)                                               │
│   ├── 元数据 (Metadata)           模板名称、类型、尺寸等        │
│   ├── 图层结构 (Layer Structure)  PSD 图层树                    │
│   └── 占位符 (Placeholders)       可替换的内容区域              │
│       ├── 图片占位符              产品图、背景图等               │
│       ├── 文本占位符              标题、卖点、价格等             │
│       └── 样式占位符              颜色、字体等                   │
│                                                                 │
│   数据源 (Data Source)                                          │
│   ├── 素材库                      用户导入的图片                 │
│   ├── 知识库                      卖点、痛点、配色               │
│   └── 用户输入                    价格、规格等                   │
│                                                                 │
│   渲染引擎 (Render Engine)                                      │
│   ├── 占位符解析                  识别模板中的占位符             │
│   ├── 数据绑定                    将数据源与占位符关联           │
│   └── 内容替换                    执行实际的替换操作             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 模板类型

### 2.1 主图模板 (MainImage)

| 属性 | 规范 |
|------|------|
| **尺寸** | 800×800 (淘宝标准) |
| **用途** | 商品列表展示主图 |
| **必需占位符** | 产品主体、背景 |
| **可选占位符** | 卖点标签、促销角标、品牌 LOGO |

```
主图模板结构示例:
├── [BG] 背景层
├── [PRODUCT] 产品主体
├── [GROUP] 卖点区域
│   ├── [TEXT:卖点1] 卖点标签1
│   ├── [TEXT:卖点2] 卖点标签2
│   └── [TEXT:卖点3] 卖点标签3
├── [BADGE] 促销角标
└── [LOGO] 品牌 LOGO
```

### 2.2 SKU 模板 (SKU)

| 属性 | 规范 |
|------|------|
| **尺寸** | 800×800 (淘宝标准) |
| **用途** | 颜色/规格选择图 |
| **必需占位符** | 产品缩略图、颜色名称 |
| **可选占位符** | 规格标签、价格 |

```
SKU 模板结构示例:
├── [BG] 背景层 (纯色/渐变)
├── [PRODUCT] 产品缩略图
├── [TEXT:颜色] 颜色名称
├── [TEXT:规格] 规格标签
└── [TEXT:价格] 价格 (可选)
```

### 2.3 详情页模板 (DetailPage)

| 属性 | 规范 |
|------|------|
| **宽度** | 750px (淘宝标准) |
| **高度** | 按模块动态计算 |
| **用途** | 商品详情页各模块 |
| **模块类型** | 见下表 |

#### 详情页模块类型

| 模块 ID | 模块名称 | 描述 |
|---------|----------|------|
| `header` | 首屏海报 | 核心卖点+促销信息 |
| `selling-points` | 卖点展示 | 3-5 个核心卖点 |
| `pain-solution` | 痛点解决 | 痛点+解决方案对比 |
| `material` | 材质说明 | 面料/材质细节 |
| `size-chart` | 尺码表 | 尺码对照表 |
| `color-display` | 颜色展示 | 全部颜色一览 |
| `detail-shots` | 细节展示 | 产品细节特写 |
| `model-shots` | 模特展示 | 上身效果图 |
| `comparison` | 对比图 | 竞品/升级对比 |
| `process` | 工艺展示 | 制作工艺流程 |
| `package` | 包装展示 | 包装/赠品 |
| `params` | 产品参数 | 规格参数表 |
| `service` | 售后服务 | 退换/保修说明 |
| `brand` | 品牌介绍 | 品牌故事/资质 |

## 3. 占位符规范

### 3.1 命名规则

占位符使用 **图层名称** 标识，格式为：

```
[TYPE:NAME:OPTIONS]

TYPE    - 占位符类型 (必需)
NAME    - 占位符名称 (必需)
OPTIONS - 附加选项 (可选，逗号分隔)
```

### 3.2 占位符类型

| 类型 | 标识 | 描述 | 示例 |
|------|------|------|------|
| **图片** | `IMG` | 可替换的图片层 | `[IMG:产品主体]` |
| **文本** | `TEXT` | 可编辑的文本层 | `[TEXT:标题]` |
| **智能对象** | `SO` | 智能对象占位 | `[SO:背景]` |
| **组/容器** | `GROUP` | 图层组占位 | `[GROUP:卖点区域]` |
| **样式** | `STYLE` | 可变样式 | `[STYLE:主题色]` |
| **重复** | `REPEAT` | 可重复的元素 | `[REPEAT:SKU项]` |

### 3.3 图片占位符

```
[IMG:NAME:fit,align,mask]

fit   - 适配模式: contain | cover | fill | none
align - 对齐方式: center | top | bottom | left | right
mask  - 是否裁剪: clip | overflow

示例:
[IMG:产品主体:cover,center,clip]
[IMG:背景:fill]
[IMG:模特图:contain,bottom]
```

### 3.4 文本占位符

```
[TEXT:NAME:source,maxlen,fallback]

source   - 数据来源: input | knowledge | auto
maxlen   - 最大字符数
fallback - 默认值

示例:
[TEXT:标题:input,20]
[TEXT:卖点1:knowledge,8,舒适透气]
[TEXT:价格:input,10,¥99]
```

### 3.5 样式占位符

```
[STYLE:NAME:property,inherit]

property - 样式属性: color | font | effect
inherit  - 是否继承: true | false

示例:
[STYLE:主题色:color]
[STYLE:标题字体:font]
```

### 3.6 重复占位符

```
[REPEAT:NAME:min,max,direction]

min       - 最小重复次数
max       - 最大重复次数
direction - 排列方向: horizontal | vertical | grid

示例:
[REPEAT:卖点标签:1,5,horizontal]
[REPEAT:SKU项:1,20,grid]
```

## 4. 图层命名规范

### 4.1 普通图层

非占位符图层使用常规命名，不包含方括号：

```
✓ 背景渐变
✓ 装饰元素
✓ 分割线

✗ [背景渐变]  ← 会被误识别为占位符
```

### 4.2 锁定图层

不可替换的图层在名称后添加 `#lock`：

```
品牌 LOGO #lock
水印 #lock
```

### 4.3 隐藏图层

默认隐藏的可选图层在名称后添加 `#hidden`：

```
[TEXT:副标题] #hidden
[IMG:装饰图] #hidden
```

### 4.4 条件图层

根据条件显示/隐藏的图层：

```
[TEXT:促销价] #if:hasPromotion
[IMG:赠品图] #if:hasGift
```

## 6. 数据绑定

### 6.1 绑定语法

在渲染时，通过 JSON 对象传递数据：

```json
{
  "product": {
    "type": "image",
    "source": "file",
    "path": "/path/to/product.png"
  },
  "title": {
    "type": "text",
    "value": "舒适透气船袜"
  },
  "sellingPoint1": {
    "type": "text",
    "value": "纯棉材质",
    "source": "knowledge",
    "knowledgeId": "sp-001"
  },
  "primaryColor": {
    "type": "style",
    "property": "color",
    "value": "#1E3A5F"
  }
}
```

### 6.2 知识库自动绑定

当占位符指定 `source: "knowledge"` 时，系统自动从知识库获取数据：

```json
{
  "sellingPoint1": {
    "source": "knowledge",
    "knowledgeType": "sellingPoint",
    "filter": {
      "category": "船袜",
      "priority": { "min": 4 }
    },
    "select": "random"
  }
}
```

### 6.3 条件渲染

根据数据条件显示/隐藏图层：

```json
{
  "conditions": {
    "hasPromotion": true,
    "hasGift": false
  }
}
```

对应模板中的条件图层：
```
[TEXT:促销价] #if:hasPromotion   ← 显示
[IMG:赠品图] #if:hasGift         ← 隐藏
```

## 8. 最佳实践

### 8.1 模板设计原则

1. **保持简洁**: 占位符不宜过多，建议 5-10 个
2. **语义清晰**: 命名应直观表达用途
3. **灵活适配**: 图片占位符应支持多种尺寸
4. **风格统一**: 同一模板包内风格应一致

### 8.2 占位符设计建议

| 场景 | 建议 |
|------|------|
| 产品图 | 使用 `contain` 适配，保持产品完整 |
| 背景图 | 使用 `cover` 适配，填满区域 |
| 卖点文案 | 限制字数 8-12 字 |
| 标题文案 | 限制字数 15-20 字 |

### 8.3 图层组织建议

```
📂 模板根
├── 📁 背景层 (锁定)
│   ├── 纯色/渐变背景
│   └── 装饰元素
├── 📁 产品层
│   └── [IMG:产品主体]
├── 📁 文字层
│   ├── [TEXT:标题]
│   └── 📁 卖点组
│       ├── [TEXT:卖点1]
│       ├── [TEXT:卖点2]
│       └── [TEXT:卖点3]
├── 📁 装饰层
│   ├── [IMG:角标] #hidden
│   └── [IMG:LOGO] #lock
└── 📁 参考层 (隐藏)
    └── 设计参考 #lock
```
