# DesignEcho 图片理解能力说明

## 一、现有能力

### 1.1 工具链

| 工具 | 作用 | 视觉模型 |
|------|------|----------|
| **getAssetPreview** | 获取图片 Base64 预览，供支持视觉的 LLM 直接「看」图 | 对话 LLM 需支持 vision |
| **analyzeAssetContent** | 用视觉模型分析图片，返回结构化描述（分类、主体、颜色、风格、建议用途） | gemini-3-flash、ollama-llava:13b、ollama-llava:7b |

### 1.2 流程

```
用户: "薄款堆堆袜文件夹里的图片做卖点分类"
  ↓
1. listProjectResources({ directory: "项目路径/薄款堆堆袜" }) → 获取文件列表
  ↓
2. 对每张图片: analyzeAssetContent({ imagePath: "完整路径" })
  ↓
3. 视觉模型返回: { category, mainSubject, colors, suggestedPlacement, ... }
  ↓
4. Agent 汇总并给出卖点分类
```

### 1.3 视觉模型调用（analyzeAssetContent）

- **位置**：`resource-manager-service.ts` → `analyzeAssetContent`
- **入口**：`resource-handlers.ts` → `resource:analyzeAsset`
- **模型顺序**：gemini-3-flash → ollama-llava:13b → ollama-llava:7b（按顺序尝试）
- **输入**：图片 Base64 + 分析 prompt（JSON 格式）
- **输出**：description、category、mainSubject、colors、style、suggestedPlacement、suggestedEffects

## 二、当前问题

### 2.1 listProjectResources 返回「没有找到图片文件」

**原因**：`listProjectResources` 未像 `searchProjectResources` 那样自动使用当前项目路径。

- `searchProjectResources`：会从 `currentProject.path` 取目录并调用 `setProjectRoot`
- `listProjectResources`：直接传 `params.directory`，若为相对路径（如 `"薄款堆堆袜"`）或未传，`scanDirectory` 会失败

**结果**：拿不到文件路径，无法调用 `analyzeAssetContent` 或 `getAssetPreview`。

### 2.2 子目录路径解析

当用户说「薄款堆堆袜文件夹」时，Agent 可能传 `directory: "薄款堆堆袜"`。  
`scanDirectory` 需要的是绝对路径，如 `项目根/薄款堆堆袜`，否则 `fs.existsSync` 会失败。

---

## 三、已实施修复

1. **listProjectResources**（`tool-executor.service.ts`）：
   - 未传 `directory` 时，自动使用 `currentProject.path` 并调用 `setProjectRoot`
   - `directory` 为相对路径（如 `"薄款堆堆袜"`）时，与项目根路径拼接后再调用 `scanDirectory`
2. **Prompt**：`enhanced-agent-prompt.ts` 已包含 `analyzeAssetContent` 的流程说明。
