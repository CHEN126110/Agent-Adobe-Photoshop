## 1. 需求概述

### 1.1 业务场景

```
电商袜子产品摄影 → 批量形态统一 → 整齐排列展示

输入:
├─ 10+ 种颜色袜子的平铺拍摄图
├─ 用户绘制的参考形状图层 (钢笔工具)
└─ 袜子类型: 短袜、中筒袜、长筒袜

输出:
└─ 所有袜子外轮廓统一到参考形状
   ├─ 内部纹理保持自然
   ├─ 花边/特殊袜口保持不变形
   └─ 无模糊、锯齿、接缝
```

### 1.2 核心要求

| 要求 | 指标 | 优先级 |
|------|------|--------|
| 速度 | < 500ms/张 | 🔴 P0 |
| 轮廓对齐精度 | < 3px 误差 | 🔴 P0 |
| 内部纹理保持 | SSIM > 0.98 | 🔴 P0 |
| 袜口保护 | 花边区域不变形 | 🔴 P0 |
| 边缘平滑度 | 无锯齿 | 🟡 P1 |

### 1.3 非目标

- ❌ 不做液化/扭曲内部纹理
- ❌ 不做 3D 重建或视角变换
- ❌ 不做 AI 图像重新生成

## 3. 核心算法

### 3.1 JFA 距离场 (Jump Flooding Algorithm)

**目的**: 高效计算每个像素到轮廓的距离

**复杂度**: O(n × log(max(w,h))) vs 暴力 O(n × 轮廓长度)

```typescript
// 算法流程
class JFADistanceField {
    compute(width: number, height: number, contour: Point2D[]): Float32Array {
        // 1. 初始化: 轮廓像素距离=0, 其他=无穷大
        const seeds = this.initializeSeeds(width, height, contour);
        
        // 2. Jump Flooding: 步长 n/2, n/4, n/8, ..., 1
        let step = Math.max(width, height) >> 1;
        while (step >= 1) {
            this.floodStep(seeds, step);
            step >>= 1;
        }
        
        // 3. 计算最终距离
        return this.computeDistances(seeds);
    }
    
    private floodStep(seeds: Int32Array, step: number): void {
        // 每个像素检查 8 个方向的邻居
        const offsets = [
            [-step, -step], [0, -step], [step, -step],
            [-step, 0],                  [step, 0],
            [-step, step],  [0, step],  [step, step]
        ];
        // 更新最近种子点
    }
}
```

**性能指标**:
- 2000×2000 图像: ~80ms (vs 暴力算法 800ms)
- 内存: 16MB (2 × width × height × 4 bytes)

### 3.2 自适应边缘带变形

**核心思想**: 只变形边缘带区域，内部完全保持

```
源图像:                    变形后:
┌──────────────────┐       ┌──────────────────┐
│    内部 (不变)    │       │    内部 (不变)    │
│  ┌────────────┐  │       │  ┌────────────┐  │
│  │            │  │       │  │            │  │
│  │  纹理保持  │  │   →   │  │  纹理保持  │  │
│  │            │  │       │  │            │  │
│  └────────────┘  │       └──│────────────│──┘
│    边缘带 (变形)  │           边缘对齐目标轮廓
└──────────────────┘
```

**算法流程**:

```typescript
class AdaptiveEdgeBandMorphing {
    async morph(
        image: Buffer,
        sourceContour: Point2D[],
        targetContour: Point2D[],
        config: MorphConfig
    ): Promise<MorphResult> {
        // 1. 计算距离场
        const distanceField = this.jfa.compute(width, height, sourceContour);
        
        // 2. 生成边缘带掩码 (距离 < bandWidth)
        const bandMask = this.generateBandMask(distanceField, config.bandWidth);
        
        // 3. 检测袜口区域
        const cuffRegion = this.cuffDetector.detect(sourceContour, image);
        
        // 4. 更新掩码 (袜口区域权重=0)
        this.applyCuffProtection(bandMask, cuffRegion);
        
        // 5. 生成自适应权重 (距离越远权重越低)
        const weights = this.generateAdaptiveWeights(distanceField, config);
        
        // 6. 计算稀疏位移场 (只在边缘带内)
        const displacement = this.computeSparseDisplacement(
            sourceContour, targetContour, bandMask
        );
        
        // 7. 应用加权位移
        return this.applyWeightedDisplacement(image, displacement, weights);
    }
}
```

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `bandWidth` | 60px | 边缘带宽度 |
| `transitionWidth` | 40px | 过渡区宽度 |
| `weightCurve` | hermite | 权重衰减曲线 |

### 3.3 智能袜口检测

**检测策略**: 多特征融合 + 保守兜底

```typescript
class SmartCuffDetector {
    detect(contour: Point2D[], image: Buffer): CuffResult {
        // 1. 位置分析 (袜口通常在上部 15-25%)
        const positionFeature = this.analyzePosition(contour);
        
        // 2. 边缘复杂度 (花边边缘高频变化)
        const complexityFeature = this.analyzeEdgeComplexity(contour);
        
        // 3. 对称性分析 (花边通常对称)
        const symmetryFeature = this.analyzeSymmetry(contour);
        
        // 4. 分类袜口类型
        const cuffType = this.classifyCuffType({
            position: positionFeature,
            complexity: complexityFeature,
            symmetry: symmetryFeature
        });
        
        // 5. 生成保护区域
        return {
            type: cuffType,  // 'plain' | 'lace' | 'double' | 'ribbed'
            region: this.getCuffRegion(contour, cuffType),
            confidence: this.getConfidence(),
            protectionMask: this.generateProtectionMask()
        };
    }
}
```

**袜口类型**:

| 类型 | 特征 | 处理方式 |
|------|------|----------|
| `plain` | 平滑边缘 | 正常变形 |
| `lace` | 高频边缘 + 对称 | 完全保护 |
| `double` | 两层结构 | 保护上层 |
| `ribbed` | 规则波纹 | 部分保护 |

### 3.4 稀疏位移场压缩

**目的**: 减少传输数据量

```typescript
interface SparseDisplacementField {
    width: number;
    height: number;
    
    // 只存储边缘带像素
    pixelCount: number;
    indices: Uint32Array;   // 像素索引 (y * width + x)
    dx: Int16Array;         // 水平位移 (×100 量化)
    dy: Int16Array;         // 垂直位移 (×100 量化)
    
    // 校验
    checksum: number;
}

// 压缩效果:
// 2000×2000 图像, 边缘带 60px
// 原始: 2000×2000×8 = 32MB
// 稀疏: ~50万像素 × 8 bytes = 4MB
// 量化: ~50万像素 × 6 bytes = 3MB
// Base64: ~4MB
// 最终: 传输减少 8x
```

## 附录 D: 架构决策记录

### D.1 UXP vs Agent 计算位置决策

**背景**：Google 模型建议 "放弃 UXP 计算，传输图像到 Agent 用 Sharp 处理"

**验证测试** (2026-01-29):

```
┌─────────────────────────────────────────────────────────────┐
│ 图像尺寸  │ Sharp+传输 │ Node.js JS │ 加速比               │
├─────────────────────────────────────────────────────────────┤
│ 2000×2000 │    111ms   │    91ms    │  0.82x              │
│ 4000×4000 │    423ms   │   299ms    │  0.71x              │
└─────────────────────────────────────────────────────────────┘
```

**关键发现**：
1. Sharp 没有原生位移场 API，需手动遍历像素
2. 传输开销 ~17ms/16MB，在可接受范围
3. 测试在 Node.js V8 中运行，无法模拟 UXP 引擎真实性能

**最终决策**：
- 采用**稀疏位移场传输方案**
- Agent 端计算位移场 + 压缩
- UXP 端接收并应用（只处理边缘带 ~20% 像素）
- 预留 Agent 全处理方案作为备选

### D.2 技术栈确认

| 组件 | 选型 | 理由 |
|------|------|------|
| 距离场 | JFA 算法 | O(n log n)，10x 加速 |
| 变形 | MLS Rigid | 保持局部形状 |
| 袜口检测 | 启发式 + 多特征 | 无需 AI 模型 |
| 传输 | 稀疏位移场 | 8x 压缩 |
| 图像处理 | Sharp (libvips) | C++ 原生性能 |
