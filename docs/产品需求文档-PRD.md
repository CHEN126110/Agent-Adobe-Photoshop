## 1. 产品概述

### 1.1 产品定位

**DesignEcho** 是一款基于 AI 的电商视觉设计智能助手，通过理解产品图片和用户需求，自动规划并生成电商所需的**主图、详情页、SKU 图**等视觉素材。

### 1.2 核心价值主张

```
┌─────────────────────────────────────────────────────────────────┐
│                      痛点 → 解决方案                             │
├─────────────────────────────────────────────────────────────────┤
│  设计师成本高         →  AI 自动化设计，降低人力依赖             │
│  设计周期长           →  从素材到成图，分钟级完成                │
│  设计质量参差不齐     →  内置设计知识库，保证专业审美水准         │
│  缺乏电商运营思维     →  融合卖点提炼、转化逻辑的智能规划         │
│  模板复用困难         →  智能模板系统，一键适配多款产品           │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 产品愿景

> 让每个电商卖家都能拥有**设计师级别的视觉表现力**，
> 同时具备**运营视角的转化思维**。

## 2. 目标用户

### 2.1 用户画像

| 角色 | 特征 | 痛点 | 核心需求 |
|------|------|------|----------|
| **中小卖家** | 1-5人团队，预算有限 | 请不起专业设计师，自己做不出效果 | 低成本出高质量图 |
| **电商运营** | 负责多店铺/多产品 | 设计产能跟不上上新速度 | 快速批量出图 |
| **初级设计师** | 1-3年经验 | 缺乏电商转化思维，排版配色不专业 | 设计指导与规范 |
| **工作室** | 承接多客户设计 | 项目多、风格杂、效率低 | 模板化、流程化 |

### 2.2 用户场景

**场景 1：新品上架**
> 卖家收到工厂寄来的新款袜子样品，拍了一组图，需要在 3 天内完成主图、详情页、SKU 图的设计并上架。

**场景 2：爆款换季**
> 去年的爆款袜子需要更新视觉，换成春季氛围，同时优化详情页的卖点表达。

**场景 3：大促活动**
> 双 11 期间需要批量给 50 个 SKU 换上活动主图，时间紧任务重。

### 3.2 设计师思维流程（Agent 需模拟）

```
1. 【看款式】这是什么产品？什么风格？目标人群是谁？
       ↓
2. 【提卖点】这个产品的核心卖点是什么？差异化在哪里？
       ↓
3. 【挖痛点】目标用户有什么痛点？这个产品怎么解决？
       ↓
4. 【分素材】哪些图适合做 SKU？哪些适合做卖点展示？
       ↓
5. 【排版面】详情页怎么分屏？信息层级怎么排？
       ↓
6. 【做主图】点击图怎么吸睛？转化图怎么说服？
       ↓
7. 【调细节】配色、字体、间距、氛围...打磨到位
```

## 6. 设计知识体系

### 6.1 知识库结构

```

├── 类目知识
│   └── 袜子类目
│       ├── 产品分类（中筒/船袜/长筒/隐形...）
│       ├── 核心卖点库（材质/功能/场景/人群...）
│       ├── 用户痛点库（起球/臭脚/勒脚/掉跟...）
│       └── 竞品视觉参考
│
├── 设计规范
│   ├── 排版原则
│   │   ├── 对齐方式（左对齐/居中/两端）
│   │   ├── 间距规范（字间距/行距/模块间距）
│   │   ├── 信息层级（标题/副标题/正文/注释）
│   │   └── 视觉动线（Z型/F型/中心辐射）
│   │
│   ├── 配色系统
│   │   ├── 主色调（品牌色/品类色）
│   │   ├── 辅助色（点缀/强调/背景）
│   │   ├── 配色方案（高级感/活力感/自然感）
│   │   └── 禁忌搭配（避免的颜色组合）
│   │
│   ├── 字体规范
│   │   ├── 标题字体（中文/英文）
│   │   ├── 正文字体
│   │   ├── 字号规范（标题/副标题/正文/标签）
│   │   └── 字重使用（粗/中/细）
│   │
│   └── 图片规范
│       ├── 构图原则（三分法/对称/留白）
│       ├── 主体占比
│       └── 背景处理
│
└── 审美标准
    ├── 设计师级审美特征
    │   ├── 克制的颜色使用
    │   ├── 精准的间距控制
    │   ├── 统一的视觉语言
    │   └── 适度的装饰元素
    │
    ├── 避免的低级错误
    │   ├── 颜色过多（>3种主色）
    │   ├── 字体混乱（>2种字体）
    │   ├── 信息堆砌
    │   └── 对齐不规范
    │
    └── 避免的过度设计
        ├── 装饰元素喧宾夺主
        ├── 过于抽象的表达
        └── 忽视产品本身
```

| D | ICON | 卖点图标 | 快速传达 | 3-6 个核心卖点 icon |
| E | KV | 氛围渲染 | 场景代入 | 使用场景、生活方式 |
| F | SKU_DISPLAY | 颜色款式展示 | 选择辅助 | 全色系一览、款式对比 |
| G | MATERIAL | 材质面料 | 品质感知 | 面料特写、工艺细节 |
| H | PAIN_POINT | 解决痛点 | 需求共鸣 | 痛点场景 → 解决方案 |
| I | OUTFIT | 穿搭展示 | 场景延伸 | 搭配建议、风格展示 |
| J | SPEC | 产品参数 | 理性决策 | 尺寸、成分、洗涤说明 |
| K | DETAIL | 细节展示 | 品质佐证 | 缝线、包边、logo 细节 |
| L | MODEL | 模特实拍 | 真实感 | 真人上身效果 |
| M | CARE | 产品维护 | 使用指导 | 洗涤保养、存放建议 |
| N | SERVICE | 售后服务 | 消除顾虑 | 退换政策、客服承诺 |

### 7.2 模块排序策略

**标准排序（转化导向）**：
```
C → B → D → H → E → G → K → L → F → I → J → M → A → N
首屏  信任  卖点  痛点  氛围  材质  细节  模特  款式  穿搭  参数  维护  营销  售后
```

**价格敏感型（低价产品）**：
```
A → C → D → F → G → J → N
营销  首屏  卖点  款式  材质  参数  售后
```

**品质导向型（中高端产品）**：
```
C → G → K → B → D → E → L → H → F → I → J → M → N
首屏  材质  细节  信任  卖点  氛围  模特  痛点  款式  穿搭  参数  维护  售后
```

### 7.3 袜子类目模块示例

| 模块 | 袜子类目具体内容 |
|------|------------------|
| **C 首屏** | 产品堆叠图 + "精梳棉 · 吸汗透气 · 四季可穿" |
| **D 卖点** | 🧵精梳棉 💨透气网眼 🦶人体工学 🔒防滑设计 |
| **H 痛点** | ❌脚臭尴尬 → ✅抗菌除臭｜❌起球掉毛 → ✅耐穿耐洗 |
| **G 材质** | 棉花特写 + 成分表（75%棉+20%涤纶+5%氨纶） |
| **K 细节** | 脚跟加固、脚尖缝合、松紧带、logo刺绣 |
| **F 款式** | 4色并排展示（黑/白/灰/藏青） |
| **J 参数** | 尺码表（均码/加大码）、厚度、适用季节 |

## 9. SKU 系统

### 9.1 SKU 组合逻辑

```
产品属性                    SKU 组合
   │                          │
   ├── 颜色 ──────────────────┼──▶ 黑色
   │   ├── 黑色               │    白色
   │   ├── 白色               │    灰色
   │   ├── 灰色               │    藏青
   │   └── 藏青               │
   │                          │
   └── 规格 ──────────────────┼──▶ × 均码
       ├── 均码               │    × 加大码
       └── 加大码             │
                              │
                              ▼
                     颜色 × 规格 = 8 个 SKU
                     
                     黑色_均码, 黑色_加大码
                     白色_均码, 白色_加大码
                     灰色_均码, 灰色_加大码
                     藏青_均码, 藏青_加大码
```

### 9.2 SKU 图设计规范

| 要素 | 规范 |
|------|------|
| **背景** | 纯白或浅灰，保持统一 |
| **产品位置** | 居中，占比 70-80% |
| **角度** | 统一角度（建议 45° 俯视） |
| **颜色标识** | 左下角色块 + 颜色名 |
| **规格标识** | 右下角规格文字 |
| **尺寸** | 800 × 800 px |
