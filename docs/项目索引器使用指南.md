# 项目索引器使用指南

## 概述

**项目索引器**是 DesignEcho 的核心功能之一，它能够将你的设计作品自动索引到 RAG 知识库，让 AI 能够学习和复用你的设计风格。

### 解决的问题

- ❌ **旧方案**：内置静态知识数据不专业，用户无法导入自己的作品
- ✅ **新方案**：直接索引用户的项目目录，自动提取设计作品并建立检索能力

---

## 快速开始

### 1. 准备你的项目目录

你的项目目录应该遵循以下结构（类似 `C-649` 样本）：

```
D:\A1 neveralone旗舰店\
├── C-649\
│   ├── 主图\          # 主图文件（800/750/1200 多尺寸）
│   ├── SKU\          # SKU 组合图
│   ├── PSD\          # PSD 源文件
│   ├── images\       # 详情页切片
│   └── 配置文件\      # CSV 配置
├── C-650\
│   └── ...
└── C-651\
    └── ...
```

### 2. 使用 UI 界面索引

1. 启动 DesignEcho Agent
2. 进入「知识库管理」页面
3. 选择「作品索引器」标签
4. 输入或选择项目根目录（例如 `D:\A1 neveralone旗舰店`）
5. 点击「扫描项目」
6. 确认扫描结果后，点击「批量索引」

### 3. 使用脚本批量索引

```bash
cd C:\UXP\2.0\DesignEcho-Agent
npx ts-node scripts/test-project-indexer.ts
```

## 功能特性

### 自动识别文件类型

索引器会自动识别以下类型的文件：

| 类型 | 目录 | 用途 |
|------|------|------|
| 主图 (`main_image`) | `主图/` | 800/750/1200 尺寸的主图 |
| 详情页切片 (`detail_page_slice`) | `images/` | 详情页每屏的切片 |
| SKU (`sku`) | `SKU/` | SKU 组合图 |
| PSD (`psd`) | `PSD/` | 设计源文件 |

### 智能描述生成

#### 方式 1：简单模式（快速）
- 使用文件名和类型生成描述
- 速度快，适合大量文件
- 不调用视觉模型

#### 方式 2：视觉模式（准确）
- 使用 VLM（Gemini 2.0 Flash）分析图片内容
- 生成语义丰富的描述
- 适合精准索引，但速度较慢

### 向量化与检索

- 使用 `bge-small-zh-v1.5` 模型生成 512 维向量
- 存储到 LanceDB 本地向量数据库
- 支持语义搜索和相似度匹配

## API 使用

### 扫描单个项目

```typescript
const result = await window.designEcho.invoke('project:scan', 'C:\\UXP\\2.0\\C-649');

if (result.success) {
    console.log('文件列表:', result.items);
}
```

### 扫描多个项目

```typescript
const result = await window.designEcho.invoke('project:scanAll', 'D:\\A1 neveralone旗舰店');

if (result.success) {
    console.log('项目映射:', result.projects);
}
```

### 索引单个项目

```typescript
const result = await window.designEcho.invoke(
    'project:index',
    'C-649',
    items,
    false  // useVision
);

if (result.success) {
    console.log(`成功: ${result.success}, 失败: ${result.failed}`);
}
```

### 批量索引项目

```typescript
const result = await window.designEcho.invoke(
    'project:indexAll',
    projectMap,
    true  // useVision
);

if (result.success) {
    console.log('索引统计:', result.stats);
}
```

### 清空项目索引

```typescript
const result = await window.designEcho.invoke('project:clearIndex', 'C-649');

if (result.success) {
    console.log(`已删除 ${result.deleted} 条记录`);
}
```

## 性能优化建议

### 1. 首次索引

- **不使用视觉模式**：适合大量项目（100+ 个）
- 索引速度：约 10-20 个文件/秒
- 后续可以针对重点项目单独启用视觉模式

### 2. 精准索引

- **使用视觉模式**：适合重点项目（10-50 个）
- 索引速度：约 2-5 个文件/秒（受 VLM 调用限制）
- 描述质量显著提升

### 3. 增量索引

- 新增项目可以单独索引，无需重新索引全部
- 使用「索引」按钮对单个项目进行索引

## 常见问题

### Q1: 索引失败怎么办？

**A**: 检查以下内容：
1. 项目目录结构是否正确
2. 文件是否损坏
3. 查看 Agent 日志 (Ctrl+Shift+I)

### Q2: 检索结果不准确？

**A**: 尝试以下方法：
1. 使用视觉模式重新索引重点项目
2. 优化查询关键词（更具体、更语义化）
3. 提供更多上下文（如"800尺寸主图 白底 居中"）

### Q3: 能否索引其他格式的文件？

**A**: 当前支持：
- 图片：`.jpg`, `.jpeg`, `.png`
- PSD：`.psd`, `.psb`

未来可扩展支持 AI、CDR 等格式。

### Q4: 索引会影响原文件吗？

**A**: **不会**。索引器只读取文件，不会修改或移动任何文件。

## 技术架构

```
┌──────────────────────────────────────────────────┐
│                     User                          │
│             (拖入项目目录)                          │
└────────────────┬─────────────────────────────────┘
                 │
                 v
┌──────────────────────────────────────────────────┐
│           ProjectIndexer                          │
│  - scanProject()    扫描项目文件                   │
│  - scanProjects()   批量扫描                       │
│  - indexProject()   索引单个项目                   │
│  - indexProjects()  批量索引                       │
└────────────────┬─────────────────────────────────┘
                 │
                 v
┌──────────────────────────────────────────────────┐
│     VLM (可选)           EmbeddingService         │
│  Gemini 2.0 Flash    bge-small-zh-v1.5           │
│  (生成图片描述)         (文本 → 向量)              │
└────────────────┬─────────────────────────────────┘
                 │
                 v
┌──────────────────────────────────────────────────┐
│              VectorStore (LanceDB)                │
│  - 512 维向量存储                                  │

│  - 支持元数据过滤                                  │
└──────────────────────────────────────────────────┘
```
