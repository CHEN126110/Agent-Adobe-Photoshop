# Eagle 知识库集成可行性分析

> 日期：2026-01-30
> 状态：可行性验证完成 ✅

## 1. 项目目标

将 Eagle 素材管理软件作为 DesignEcho 的设计知识库，实现：
- 语义化素材搜索（"找暖色调的背景图"）
- 设计参考智能推荐
- 减少 AI 模型的"幻觉"问题

## 2. Eagle API 验证结果

### 2.1 API 可用性 ✅

Eagle 在本地运行时提供 HTTP API：
- **端口**：41595
- **协议**：HTTP REST
- **状态**：正常可用

### 2.2 已验证的接口

| 接口 | 方法 | 用途 | 状态 |
|------|------|------|------|
| `/api/application/info` | GET | 获取应用信息 | ✅ |
| `/api/library/info` | GET | 获取库信息 | ✅ |
| `/api/folder/list` | GET | 获取文件夹列表 | ✅ |
| `/api/item/list` | GET | 获取素材列表 | ✅ |

### 2.3 素材元数据结构

```json
{
  "id": "ML0AMUN46AGSO",
  "name": "素材名称",
  "ext": "jpg",
  "width": 790,
  "height": 1298,
  "size": 642626,
  "tags": ["标签1", "标签2"],
  "folders": ["LQBZIFJSWBQUY"],
  "url": "来源URL",
  "annotation": "备注/描述",
  "palettes": [
    { "color": "140 157 173", "ratio": 46 },
    { "color": "85 123 161", "ratio": 14 }
  ]
}
```

### 2.4 关键数据点

| 字段 | 类型 | 用途 |
|------|------|------|
| `id` | string | 唯一标识符 |
| `name` | string | 素材名称（可用于文本搜索） |
| `ext` | string | 文件格式 |
| `width/height` | number | 尺寸信息（可筛选横版/竖版） |
| `tags` | array | 用户标签（核心语义信息） |
| `folders` | array | 分类信息 |
| `palettes` | array | **自动提取的调色板**（非常有价值！） |
| `annotation` | string | 用户备注 |

### 3.2 向量化策略

| 数据类型 | 向量化方法 | 模型建议 |
|----------|------------|----------|
| 图片内容 | 图像嵌入 | CLIP / SigLIP |
| 素材名称 | 文本嵌入 | BGE-M3 / text-embedding-3 |
| 标签 | 文本嵌入 | 同上 |
| 调色板 | 颜色语义嵌入 | 自定义映射 |

### 3.3 查询流程

```
用户: "找一张适合母婴产品的暖色调背景"
         ↓
    [查询理解]
         ↓
    分解为: 
      - 颜色: 暖色调 (黄、橙、粉)
      - 用途: 背景图
      - 风格: 母婴/温馨
         ↓
    [向量检索]
         ↓
    返回 Top-5 匹配素材 + 预览图
```

## 5. 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| Eagle 未运行时 API 不可用 | 中 | 中 | 添加连接状态检测，优雅降级 |

## 7. Eagle 能解决什么问题？

### 🎯 核心结论：**同时支持文案和排版**

基于实际数据分析，Eagle 知识库可以：

### 7.1 排版问题 ✅

| 能力 | 数据支撑 | 实现方式 |
|------|----------|----------|
| **版式参考** | 详情页排版、细节展示排版、板式1 等 PSD | 按设计意图检索相似版式 |
| **首屏设计** | C 详情页首屏 文件夹 | 图像相似度匹配 |
| **产品展示布局** | F 颜色款式展示、E KV图 | 按颜色数量/风格检索 |
| **配色方案** | 100% 素材有 palettes 数据 | 颜色向量匹配 |

### 7.2 文案问题 ✅

| 能力 | 数据支撑 | 实现方式 |
|------|----------|----------|
| **营销文案** | A 营销信息、文案合集.psd | OCR 提取 + 文本向量化 |
| **产品卖点** | H 解决痛点问题 | 按产品类型检索 |
| **品牌背书** | B 信任状/品牌背书 | 结构化知识库 |
| **产品参数** | 产品参数 文件夹 | 模板化提取 |

### 7.3 局限性 ⚠️

| 问题 | 当前状态 | 建议解决方案 |
|------|----------|--------------|
| **标签缺失** | 仅 1% 素材有标签 | 使用 CLIP 自动生成标签 |
| **文案需 OCR** | 文案在图片中 | 集成 OCR 服务提取文字 |
| **模板未结构化** | PSD 作为整体存储 | 需要额外的模板解析系统 |

## 8. 结论

### 可行性评估：✅ 可行

**优势**：
1. Eagle API 接口完善，可以获取完整的素材元数据
2. `palettes` 调色板数据对颜色搜索非常有价值
3. 架构清晰，可以渐进式实现
4. **已有文案和排版素材**，不需要从零开始

**建议**：
1. 先实现 Phase 1 基础连接，验证端到端流程
2. 使用本地 CLIP 模型进行向量化（已有 ONNX 经验）
3. 初期专注于"颜色搜索 + 版式参考"场景
4. 后续引入 OCR 提取图片中的文案
