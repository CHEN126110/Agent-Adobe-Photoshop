# 选区抠图功能现状

## 概述

选区抠图是 DesignEcho 的核心功能之一，允许用户通过绘制选区来精确分割图像中的特定对象。该功能类似于 Photoshop 的"对象选择工具"，但使用 AI 模型进行智能分割。

---

## 当前架构

### 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|

## 当前实现流程

```
用户绘制选区
    ↓
UXP 获取选区边界框 + 图层图像
    ↓
发送到 Agent (WebSocket)
    ↓
SAM2.1 识别主体 (Box Prompt)
    ↓
计算 SAM 主体边界框
    ↓
裁剪图像 (基于 SAM 边界)
    ↓
BiRefNet 边缘细化
    ↓
融合蒙版 (SAM + BiRefNet)
    ↓
应用用户选区约束 (AND 操作)
    ↓
返回蒙版到 UXP
    ↓
应用图层蒙版
```

## 核心问题诊断（已验证）

### 问题 1: BiRefNet 角色误用 ⚠️ 根本原因

**事实验证**：
- BiRefNet 是 **SOD (Salient Object Detection)** 模型
- 它的训练目标是"找出画面里最显眼的东西"
- **它不接受用户指令，按自己的判断分割**

**问题表现**：
```
场景：用户想选择绿幕上的一个配角（角落物体）

1. 你裁切了这个区域
2. BiRefNet 看着裁切图，判断"哪个最显眼"
3. BiRefNet 可能认为"背景绿幕纹理更显眼"或"某个反光更显眼"
4. 生成的 Mask 与用户意图不符
5. 融合时两者边缘不重合 → 背景残留、重影
```

**代码验证**：
- `src/main/services/matting-service.ts` 调用 BiRefNet 时无法传入用户约束
- BiRefNet 输入仅有图像，无 Trimap/Prompt 输入

### 问题 2: SAM 输出分辨率缺陷 ⚠️ 已确认

**代码验证**（`sam-service.ts`）：

```typescript
// 第 472-474 行：蒙版输入是 256x256
feeds[name] = new this.ort.Tensor(
    'float32',
    new Float32Array(256 * 256).fill(0),
    [1, 1, 256, 256]
);

// 第 526 行：输出形状确认
// SlimSAM 输出形状: [1, 1, 3, 256, 256] (batch, query, num_masks, H, W)
```

**问题表现**：
- SAM 原生输出仅 **256×256**
- 放大到 800×800 或更高分辨率时，每个像素被拉伸 3-4 倍
- 边缘必然出现"马赛克"或锯齿

**当前处理方式**：
```typescript
// 第 567-617 行：双三次插值放大
// 虽然使用了 Bicubic，但根本问题是信息量不足
for (let dstY = 0; dstY < originalHeight; dstY++) {
    // ... Catmull-Rom 双三次插值
}
```

### 问题 3: 融合策略冲突

**当前融合逻辑**：
```typescript
// SAM 认为是前景（>64）→ 使用 BiRefNet 边缘细节
// SAM 边缘区域（16-64）且 BiRefNet 有内容（>64）→ 信任 BiRefNet
// SAM 认为是背景（<16）但 BiRefNet 高置信度（>220）→ 仍信任 BiRefNet
```

**问题**：
- 如果 BiRefNet 把背景识别为主体，融合会把背景"混入"结果
- SAM 和 BiRefNet 的边缘往往不重合，导致"越修越乱"

## 专业解决方案

### 路径 A: SAM-HQ (高精度替代方案) ⭐ 推荐优先

**原理**：SAM-HQ 在 SAM 内部增加 High-Quality Token，专门解决 256x256 边缘模糊问题

**优势**：
- 直接输出 **1024×1024** 高精度蒙版
- 边缘极其锐利
- 可**完全移除 BiRefNet**，简化流程

**改动**：
1. 导出 SAM-HQ ONNX（官方提供脚本）
2. 替换现有 SAM2.1-Large
3. 移除 BiRefNet 融合逻辑

**适用场景**：硬边选择（产品、车、人）

**资源**：
- 官方仓库: https://github.com/SysCV/sam-hq
- 导出脚本: `python scripts/export_onnx_model.py`

### 路径 B: Trimap + Alpha Matting (工业级方案)

**适用场景**：毛发、毛绒玩具、半透明物体

**新流程设计**：

```
1. SAM 预测 → 得到粗糙 Mask
       ↓
2. 生成 Trimap（三分图）:
   - 腐蚀 (Erode) → 绝对前景（白色）
   - 膨胀 (Dilate) → 绝对背景（黑色）
   - 膨胀 - 腐蚀 = 未知区域（灰色）
       ↓
3. Matting 模型（ViTMatte/ModNet）:
   - 输入: 原图 + Trimap
   - 输出: 精准 Alpha 通道
```

**关键点**：
- **放弃 BiRefNet**（它不接受 Trimap 输入）
- 引入 ViTMatte 或 ModNet (ONNX版本)

**可用模型**：
- ViTMatte: https://hugging-face.cn/docs/transformers/model_doc/vitmatte
- PP-MattingV2 (PaddleSeg): 已有 ONNX 版本
- RMBG-2.0: 90.14% 准确率

## 急救措施（快速修复）

### 1. 修正融合策略（关键）

**当前危险逻辑**：
```typescript
// 问题：BiRefNet 把背景识别为主体时，会混入结果
if (samValue > 16 && birefValue > 64) → 信任 BiRefNet
```

**建议改为"交集优化法"**：
```typescript
// 1. SAM Mask 膨胀 20px 得到 ROI
// 2. Final_Mask = 0 (默认全黑)
// 3. 在 ROI 内，取 BiRefNet_Mask 值
// 4. 在 ROI 外，强制为 0
// 效果：强迫 BiRefNet "闭嘴"，只在 SAM 圈定范围内说话
```

### 2. 改进 SAM 锯齿处理

**当前方法**：仅使用 Bicubic 插值

**建议增加后处理**：
```typescript
// 1. 获取 256x256 Mask
// 2. Resize 到目标尺寸（Bicubic）
// 3. 应用高斯模糊 (sigma=2~3)
// 4. 应用阈值截断 (pixel > 128 ? 255 : 0)
// 效果：模糊的边缘变回锐利，但比直接 resize 更平滑圆润
```

### 3. 坐标映射检查

**可能的 Bug**：裁剪/还原坐标偏移导致背景残留

**检查点**：
```typescript
// 裁剪
Crop_Image = Original_Image[y:y+h, x:x+w]

// 推理
Mask_Local = BiRefNet(Crop_Image)

// 还原
Mask_Global[y:y+h, x:x+w] = Mask_Local

// Bug 点：如果 x, y 偏移算错，Mask 会盖在错误位置
```

**验证方法**：日志打印 `Crop Box` 和 `Paste Box` 坐标

## 模型特性对比

| 特性 | SAM2.1-Large | SAM-HQ | BiRefNet | ViTMatte |
|------|-------------|--------|----------|----------|
| 类型 | 交互式分割 | 高精度交互分割 | 显著性分割 | Alpha Matting |
| 输入 | Box/Point Prompt | Box/Point Prompt | 完整图像 | 图像 + Trimap |
| 输出分辨率 | 256×256 | **1024×1024** | 1024×1024 | 原图尺寸 |
| 边缘质量 | 差（需放大） | **优秀** | 良好 | **毛发级** |
| 主体识别 | 精准（Prompt） | 精准（Prompt） | 自动（显著性） | 依赖 Trimap |
| 局部分割 | ✅ 支持 | ✅ 支持 | ⚠️ 需裁剪 | ✅ 支持 |

## 已实施的改进（2026-01-25）

### 1. SAM 蒙版后处理

在 SAM 蒙版放大后添加了后处理流程：

```typescript
// 高斯模糊 (sigma=2.5, 7x7 核)
// → 消除锯齿和块状伪影

// 阈值截断 (threshold=128)
// → 将模糊边缘变回锐利
```

### 2. 交集优化法融合策略

**核心改进**：强迫 BiRefNet "闭嘴"，只在 SAM 圈定范围内说话

```typescript
// Step 1: 对 SAM Mask 膨胀 20px 创建 ROI
// Step 2: 在 ROI 内 → 使用 BiRefNet 边缘
// Step 3: 在 ROI 外 → 强制为 0（消除背景错误识别）
```

**对比**：
- 旧策略：即使 SAM 不认可，BiRefNet 高置信度也保留 → 背景残留
- 新策略：BiRefNet 只在 SAM 边缘区域生效 → 精确控制
