## 一、从素材库置入图片

### 1.1 问题根因

- **UXP 沙箱限制**：`fs.getEntryWithUrl(fileUrl)` 对项目路径（如 `D:\A1 neveralone旗舰店\...`）访问常失败
- **日志证据**：`getEntryWithUrl 失败: Could not find an entry of 'file:///D:...'`
- **现有流程**：Agent `searchProjectResources` → 得到 `filePath` → UXP `placeImage({ filePath })` → 失败

### 1.2 已实现方案

| 层级 | 修改 | 说明 |
|------|------|------|
| **UXP** | `place-image.ts` | 支持 `imageData`（Base64）：写入临时文件 → `placeEvent` |
| **Agent** | `tool-executor.service.ts` | `placeImage` 收到 `filePath` 时，先调用 `readImageBase64`，转为 `imageData` 再发 UXP |
| **兼容** | `place-image.ts` | 支持 `base64` 别名（main-image 技能使用） |

### 1.3 流程

```
用户: "把素材库里的背景图置入"
  → Agent: searchProjectResources("背景")
  → Agent: placeImage({ filePath: "D:\...\bg.jpg" })
  → Tool Executor: readImageBase64(filePath) → { base64 }
  → Tool Executor: placeImage({ imageData: base64, imageFormat: "jpeg" })
  → UXP: 写临时文件 → placeEvent → 置入成功
```

### 1.4 验证要点

- 项目路径（含中文、空格）可正常置入
- 支持格式：jpg、png、gif、webp、tiff、bmp
- 主图技能 AI 背景置入（base64）继续可用

## 二、图层准确快速理解

### 2.1 现有能力

| 工具 | 输出 | 用途 |
|------|------|------|
| `getLayerHierarchy` | 树形结构、父子关系、类型统计、bounds 可选 | 理解图层组织 |
| `getElementMapping` | 位置、大小、类型、文本、位置描述 | 布局分析、元素定位 |
| `getCanvasSnapshot` | 画布 Base64 快照 | AI 视觉分析 |
| `getLayerProperties` | 单图层详细属性 | 精细操作 |
| `getLayerBounds` | 边界框 | 位置计算 |

### 2.2 「准确」优化方向

1. **语义标注**：在 `getElementMapping` 或 `getLayerHierarchy` 中增加启发式标签
   - 如：名称含「主图」「SKU」「占位」「图片」→ 标注 `role: 'product' | 'placeholder' | 'background'`
   - 需与业务约定命名规范

2. **与 AI 视觉结合**：`getCanvasSnapshot` + 视觉模型 → 生成「主图区域」「文字块」等语义
   - 已有 `analyzeLayout` 等能力，可扩展 prompt

3. **类型细化**：`getLayerHierarchy` 的 `kind` 已较细（text、pixel、smartObject、adjustment 等），可补充：
   - 形状图层（shape）
   - 填充图层（fill）

### 2.3 「快速」优化方向

1. **合并调用**：提供 `getDocumentContext` 一类聚合工具
   - 一次返回：hierarchy 摘要 + elementMapping 精简版 + 文档信息
   - 减少 Agent 多轮 `getLayerHierarchy` + `getElementMapping` 调用

2. **按需 bounds**：`getLayerHierarchy` 的 `includeBounds` 默认 false，保持现状
   - 需要位置时再调 `getLayerBounds` 或 `getElementMapping`

3. **缓存**：Agent 端对同一文档的 hierarchy/mapping 做短期缓存（如 5 秒），避免重复请求

### 2.4 建议优先级

| 优先级 | 项 | 工作量 | 收益 |
|--------|-----|--------|------|
| P1 | 合并 `getDocumentContext` | 中 | 减少调用、加快理解 |
| P2 | 语义标注（基于名称关键词） | 低 | 提升「准确」 |
| P2 | 缓存 | 低 | 提升「快速」 |
| P3 | AI 视觉语义扩展 | 高 | 更智能理解 |
